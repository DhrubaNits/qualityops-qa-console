# Definition of Ready

A requirement is ready for QA test design when it has:

* Clear business objective
* Clear user role or actor
* Clear workflow entry point
* Clear expected outcome
* Complete acceptance criteria
* Positive scenarios
* Negative scenarios
* Validation rules
* Error message expectations
* Data setup or preconditions
* Integration and dependency details
* Non-functional requirements where applicable
* Audit or logging expectations for critical workflows
* Clear pass/fail conditions

A requirement needs QA review if any of the following are missing:

* Description
* Acceptance criteria
* Expected result
* User role
* Validation behavior
* Error handling
* Integration behavior
* Non-functional expectation
